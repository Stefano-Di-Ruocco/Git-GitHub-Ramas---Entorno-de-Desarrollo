# Historia de Git

## Origen y Creación

Git fue creado por Linus Torvalds en abril de 2005. Torvalds es un 
ingeniero de software finlandés, nacido el 28 de diciembre de 1969 en Helsinki, 
conocido principalmente por ser el creador del kernel de Linux.

## ¿Por qué se creó Git?

Durante la mayor parte del mantenimiento del kernel de Linux (1991-2002), los 
cambios en el software se realizaban mediante parches y archivos enviados por 
correo electrónico.

En 2002, el proyecto comenzó a utilizar BitKeeper, un sistema de control de 
versiones propietario. Sin embargo, en 2005 BitKeeper dejó de ofrecer su 
licencia gratuita, lo que obligó a Torvalds a crear su propia herramienta.

Git fue desarrollado en tan solo unas pocas semanas.

## Objetivos de Git

1. Velocidad - Operaciones rápidas
2. Diseño sencillo - Fácil de entender
3. Soporte para desarrollo no lineal - Miles de ramas paralelas
4. Completamente distribuido - Cada desarrollador tiene una copia completa
5. Eficiencia - Capaz de manejar grandes proyectos

## ¿Para qué sirve Git hoy?

Git es el sistema de control de versiones más utilizado en el mundo:

- Desarrollo de software colaborativo
- Historial completo de cambios
- Trabajo con ramas y fusiones
- Respaldo distribuido
- Integración con GitHub, GitLab y Bitbucket
